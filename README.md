# iterm2-status-bar

One glance at any iTerm2 tab tells you **what that Claude Code session is building, where it's at, and what's next** — a single ~15-word status line, generated by Haiku, updated automatically after every turn.

```
🤖 watcher rework in progress · next: test watcher behavior
```

Built for the tab-switcher's pain: seven Claude sessions open, every tab looks the same, and you've forgotten where session four left off.

## The status line

Two facts tracked, one displayed, ~15 words — every word competes for its slot:

| Part | Shape | Behavior |
|------|-------|----------|
| **GOAL** | 2–4 word label | The high-level thing being built. Sticky: repeated verbatim on every update unless it clearly misdescribes the session. Sessions often open with an exploratory ask ("explain this repo") before the real goal emerges — once it does, the label corrects itself, then holds stable. Revisions are logged. **Not displayed** (tab titles already identify sessions) — it anchors generation and pivot detection. |
| **THRUST** | `<state> · next: <step>` | The displayed line: where work stands right now plus the obvious next step, regenerated after each turn from the agent's own end-of-turn report. `· next:` is omitted when there's no clear next step. |

## How it works

```
Claude Code hooks                     iTerm2
─────────────────                     ──────
UserPromptSubmit ─┐
Stop ─────────────┼─► lib/notebar.py ─► claude -p --model haiku ─► user.note ─► status bar
SessionStart ─────┘        │                (~600 tokens in,          (per-session
                           ▼                 ~25 out, async)           variable)
              ~/.claude/statusbar/<session_id>.json
```

- **UserPromptSubmit** — first prompt of a session generates the initial GOAL label (from the prompt text alone, no transcript read). Later prompts just store an excerpt for pivot detection and append `⋯` to the displayed line to mark a turn in progress. No LLM call.
- **Stop** — the real update. Tail-reads the transcript (last ~256 KB, never the whole file), extracts the last assistant text message, and feeds *goal + previous status + latest prompt excerpt + that message* to Haiku. This is also where the GOAL label evolves: each update may revise it when it no longer describes what the session is actually building (revisions logged as `goal revised: 'x' -> 'y'`), so an off-target initial label — or a session that pivots — self-corrects within a turn. Hook exits instantly; generation runs in the background under a per-session lock.
- **SessionStart** — re-emits the stored line on `--resume` so a resumed session labels its new tab immediately. Also prunes state files older than 14 days.

Display is iTerm2's built-in **Interpolated String** status bar component rendering `🤖 \(user.note?)`; the hook sets `user.note` per-session via the iTerm2 Python API, so each tab shows its own line.

### Token cost

One Haiku call per completed turn (plus one at session start): roughly 600 input / 25 output tokens of prompt content. The transcript is never pushed into the subprocess — inputs are the four small strings above.

## Install

Requirements: macOS, iTerm2 with the Python API enabled (**Settings → General → Magic → Enable Python API**), the `iterm2` Python package (`pip3 install iterm2`), and the `claude` CLI on PATH.

1. Clone this repo (paths below assume `~/work/iterm2-status-bar`).

2. Add the status bar component to your profile — run **from inside an iTerm2 tab** using that profile:

   ```sh
   python3 setup/add-statusbar-component.py
   ```

   Idempotent; skips if a `user.note` component already exists. (Prefer clicking? Settings → Profiles → Session → Status bar enabled → Configure Status Bar → add **Interpolated String** with value `🤖 \(user.note?)`.)

3. Wire the hooks into `~/.claude/settings.json`:

   ```json
   {
     "hooks": {
       "UserPromptSubmit": [{ "matcher": "", "hooks": [{ "type": "command", "command": "~/work/iterm2-status-bar/hooks/note-prompt.sh" }] }],
       "Stop":             [{ "matcher": "", "hooks": [{ "type": "command", "command": "~/work/iterm2-status-bar/hooks/note-stop.sh" }] }],
       "SessionStart":     [{ "matcher": "", "hooks": [{ "type": "command", "command": "~/work/iterm2-status-bar/hooks/note-session-start.sh" }] }]
     }
   }
   ```

   Merge into existing hook arrays if you have them. Hooks are snapshotted when a session starts — **already-running sessions won't pick these up until restarted**.

4. Start a new Claude Code session and send a prompt. The label appears within seconds; the full status line lands after the first completed turn.

## Configuration

Environment variables, settable in `settings.json` hook commands or your shell profile:

| Variable | Default | Purpose |
|----------|---------|---------|
| `NOTEBAR_MODEL` | `haiku` | Model for `claude -p` generation calls |
| `CLAUDE_BIN` | `claude` on PATH | Explicit path to the claude CLI |

Tunables at the top of `lib/notebar.py`: `MAX_WORDS` (status budget, 15), `PREFIX_MAX_WORDS` (goal label cap, 4), `CLAUDE_TIMEOUT` (seconds, 120).

## Recursion & side-effect guard

Every generation call runs with `ITERM_NOTE_HOOK=1`. All three hooks exit immediately when it's set, so the nested `claude -p` can never trigger another generation. Any *other* hook that shouldn't fire for these throwaway sessions (notification sounds, bells, loggers) can opt out with one line:

```sh
[ -n "$ITERM_NOTE_HOOK" ] && exit 0
```

## Troubleshooting

- **Nothing in the status bar** — is the session's tab using the profile you added the component to? Is the Python API enabled? Check `~/.claude/statusbar/notebar.log`.
- **Line stuck on an old state** — the Stop hook only fires on completed turns; a session that predates hook installation never updates (restart it). Overlapping turns skip updates (`locked, skipping` in the log) and catch up on the next turn.
- **Wrong goal label** — it self-corrects on the next update once the transcript makes the real goal visible; revisions are logged as `goal revised: 'x' -> 'y'`.
- **State/inspection** — per-session JSON lives at `~/.claude/statusbar/<session_id>.json` (`goal`, `status`, `last_prompt`, `updated`).

## Uninstall

Remove the three hook entries from `~/.claude/settings.json`, delete the Interpolated String component from your profile's status bar, and `rm -rf ~/.claude/statusbar`.
